# 第2步: MCTS 树搜索推演（核心机制）

> **一句话**: 把每个候选方案的执行路径建模为一棵树，通过多轮"选择→展开→模拟→反向传播"迭代，逐步收敛到最优决策。
> **这与围棋 MCTS 一致**——不是评估一次就定论，而是反复探索不同路径，价值估计越来越准。

---

## 核心理念：从"一次性评估"到"树搜索"

```
旧版（1.3.0 之前）:
  方案A → Step1→Step2→Step3 → V=0.88（一次性，不再回头）
  方案B → Step1→Step2→Step3 → V=0.81
  问题: 一步推到底，没回头检查，不知道"如果 Step2 失败了"会怎样

新版（1.4.0）:
  方案A - 执行路径树:
      Root
       ├── Step1-成功 (n=3, w=2.8, V=0.93)
       │    ├── Step2-路径a (n=2, w=1.6, V=0.80)
       │    │    └── Step3-完成 (n=1, w=0.7, V=0.70)
       │    └── Step2-路径b (n=1, w=0.9, V=0.90) ← 后来发现更好的分支
       └── Step1-失败 (n=1, w=0, V=0.00) ← 探索了失败路径
           └── 备用方案 (n=1, w=0.6, V=0.60)

  每轮迭代:
    ① Selection: 从 Root 沿 UCB 最大的路径向下走到叶节点
    ② Expansion: 在叶节点展开一个新分支（新的可能方向）
    ③ Simulation: 从这个新分支快速推演到底
    ④ Backpropagation: 把结果反向传播更新路径上所有节点
```

## MCTS 四阶段详解

### 阶段总览

```
┌─────────────────────────────────────────────────────────────────┐
│                  MCTS 树搜索 —— 一轮迭代                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ① SELECTION（选择）                                            │
│     从 Root 开始，每层用 UCB 选择最优子节点，一直走到叶节点         │
│     UCB(node) = V_node + c × √(ln(N_parent) / n_node)           │
│                                                                 │
│  ② EXPANSION（展开）                                            │
│     在到达的叶节点上，展开一个新的子节点                            │
│     子节点 = 下一步可能的方向（成功路径、失败路径、备用方案等）       │
│                                                                 │
│  ③ SIMULATION（模拟）                                           │
│     从新节点快速推演到终点                                        │
│     得到该路径的最终价值 V_leaf                                    │
│                                                                 │
│  ④ BACKPROPAGATION（反向传播）                                  │
│     将 V_leaf 沿路径反向传播，更新每个祖先节点的 w, n, V           │
│     n_new = n_old + 1                                           │
│     V_new = V_old + (V_leaf - V_old) / n_new                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 树节点结构

每个节点存储以下信息（模拟围棋 MCTS 的节点状态）：

```
节点 = {
    id: "方案A-Step1-成功",
    description: "基于 gin-jwt 扩展 access token 中间件",
    parent: "方案A-Root",           // 父节点
    children: ["节点id1", ...],     // 子节点列表
    
    // MCTS 统计量
    n: 0,                           // 访问次数
    w: 0.0,                         // 累计价值（所有模拟的和）
    V: 0.0,                         // 平均价值 = w / n
    σ²: 0.0,                        // 价值方差
    
    // 节点属性
    node_type: "ACTION",            // ACTION | RISK | FALLBACK | TERMINAL
    is_terminal: false,             // 是否是终止节点（执行完成）
    expansion_potential: "HIGH",    // 还有多少可展开的方向 HIGH | MED | LOW | NONE
}
```

## ① SELECTION — UCB 驱动的路径选择

### 核心公式

```
UCB(child) = V_child + c × √(ln(N_parent) / n_child)

其中:
  V_child   = 子节点的平均价值（w/n）
  N_parent  = 父节点的总访问次数
  n_child   = 子节点的访问次数
  c         = 探索常数（默认 √2 ≈ 1.414）

规则:
  - n_child = 0 的节点（未被访问过）→ UCB = +∞（优先探索）
  - V_child 高的节点 → 利用（Exploitation）
  - n_child 小的节点 → 探索（Exploration）
  - c 越大 → 越倾向探索；c 越小 → 越倾向利用
```

### Selection 执行规则

```
从 Root 开始向下选择，直到到达一个"可展开"的节点:

第1步: 从 Root 开始
  current = Root

第2步: 当 current 不是叶节点且有子节点时:
  ┌─ 如果 current 还有未展开的潜在方向:
  │   → 停止 Selection，进入 Expansion
  ├─ 如果 current 所有子节点都已被探索:
  │   → 选择 UCB 最大的子节点
  │   → current = 该子节点
  └─ 如果 current 是终止节点:
      → 停止 Selection（已达到终点）

第3步: 到达可展开节点时:
  → 输出 selection_path: [Root, Node_a, Node_b, ..., current]
  → 进入 Expansion
```

### UCB 选择示例

```
Root (N=5)
├── 方案A (n=3, V=0.80)
├── 方案B (n=1, V=0.90)  
├── 方案C (n=1, V=0.60)
└── 方案D (n=0) ← 未探索，UCB=+∞，优先选择

UCB 计算（c=√2=1.414）:
  UCB(方案A) = 0.80 + 1.414 × √(ln(5)/3) = 0.80 + 0.73 = 1.53
  UCB(方案B) = 0.90 + 1.414 × √(ln(5)/1) = 0.90 + 1.79 = 2.69
  UCB(方案C) = 0.60 + 1.414 × √(ln(5)/1) = 0.60 + 1.79 = 2.39
  UCB(方案D) = +∞ (未探索) ← 本轮选方案D

第2轮后（方案D探索完，V=0.70）:
  Root (N=6)
  UCB(方案A) = 0.80 + 1.414 × √(ln(6)/3) = 0.80 + 0.73 = 1.53
  UCB(方案B) = 0.90 + 1.414 × √(ln(6)/1) = 0.90 + 1.89 = 2.79 ← 选B
  UCB(方案C) = 0.60 + 1.414 × √(ln(6)/1) = 0.60 + 1.89 = 2.49
  UCB(方案D) = 0.70 + 1.414 × √(ln(6)/1) = 0.70 + 1.89 = 2.59
```

## ② EXPANSION — 展开新分支

### 展开规则

在 Selection 到达的节点处，展开一个新的子节点（下一步的可能方向）。

```
展开方向 = 当前节点的下一步可能走向:

ACTION 节点的展开方向:
  1. 成功路径: "第一步按预期成功，进入第二步"
  2. 部分成功: "第一步大部分成功，但有一个小问题需要修复"
  3. 遇到障碍: "第一步遇到技术障碍，需要调整方案"

RISK 节点的展开方向:
  1. 风险规避成功: "备用方案生效，绕过了风险"
  2. 风险发生: "风险确实发生了，需要处理后果"
  3. 风险误判: "这个风险实际上不存在/影响很小"

FALLBACK 节点的展开方向:
  1. 备用方案生效: "切换到备用方案，继续执行"
  2. 备用方案失败: "备用方案也失败，需要回滚"
  3. 放弃当前路径: "此路不通，回到上一级重新选择"

TERMINAL 节点:
  → 不再展开（已是终点）
```

### 展开时机的动态调整

```
根据节点的 expansion_potential 决定展开深度:

expansion_potential = HIGH:
  → 展开 2~3 个新的子节点（该节点还有很多未探索方向）

expansion_potential = MED:
  → 展开 1 个新的子节点

expansion_potential = LOW:
  → 展开 1 个"快速检查"子节点（简化的方向验证）

expansion_potential = NONE:
  → 该节点已穷尽所有方向，不再展开
```

## ③ SIMULATION — 快速推演到底

### 与旧版 Step1→Step2→Step3 的区别

```
旧版: 对整个方案做完整的3步因果链推演（重、慢、一次性）
新版: 从新展开的节点快速模拟到底（轻、快、可重复多次）

关键差异:
  - 旧版一次推演 = 从根到底的完整因果链
  - 新版一次模拟 = 从当前节点到底的快速 roll-out
  - 新版多轮模拟互相补充 → 价值估计更准确
```

### 模拟规则

```
从 Expansion 创建的新节点开始，快速推演到终点:

模拟步骤:
  1. 识别当前节点的状态: 在什么位置？已知哪些信息？
  2. 向前看一步: 从这个位置出发，下一步最可能发生什么？
  3. 继续向前: 再下一步？
  4. 直到终点: 到达 TERMINAL（执行完成/失败确定/需要回滚）

模拟深度控制:
  - 简单任务: 1~2 步到底
  - 中等任务: 2~3 步到底
  - 复杂任务: 3~4 步到底

模拟输出:
  V_leaf = 模拟路径的最终价值 (0.0 ~ 1.0)
  + 简短的理由说明

模拟后不修改树结构:
  - 模拟路径上的中间节点不加入树（除非后续 Expansion 创建它们）
  - 只有反向传播会更新已有节点的统计量
```

## ④ BACKPROPAGATION — 价值反向传播

### 核心公式

```
从叶节点沿 selection_path 反向传播:

for node in reversed(selection_path + [new_node]):
    node.n += 1
    node.w += V_leaf
    node.V = node.w / node.n
    
    # Welford 在线方差更新
    if node.n == 1:
        node.σ² = 0.0
    else:
        delta = V_leaf - node.V_before
        delta2 = V_leaf - node.V
        node.M2 += delta × delta2
        node.σ² = node.M2 / node.n
```

### 反向传播的影响

```
一轮推演后:

推演了: 方案A → Step1-成功 → Step2-路径b（之前未探索）
模拟结果: V_leaf = 0.85

反向传播:
  方案A-Step2-路径b: n=1, V=0.85 ← 新节点，初值
  方案A-Step1-成功:   n: 3→4, V: 0.93→0.91 ← 被拉低了一点
  方案A-Root:         n: 4→5, V: 0.82→0.83 ← 稍微提升

下一轮 Selection 时:
  方案A-Step2-路径b 的 UCB 很高（n=1, V=0.85）→ 可能被再次选中验证
  方案A-Step1-成功 的 V 被拉低 → UCB 下降 → 其他子节点获得更多探索机会
```

## 迭代控制

### 迭代次数

```
迭代不是无限进行的，由以下条件共同决定:

硬上限（防止无限循环）:
  - 简单任务: 最多 5 轮
  - 中等任务: 最多 10 轮
  - 复杂任务: 最多 20 轮
  - 调试任务: 最多 8 轮

收敛条件（满足任意一条就停止）:
  ① 最优方案的 V 估计稳定了:
     最近 3 轮迭代中，Root 下最优子节点的 V 变化 < 0.05
     → "再迭代也不会改变结论了"

  ② 所有高价值节点的 n 都 ≥ 3:
     访问次数足够多，估计已经很准确
     → "已经充分探索了"

  ③ Root 下最优方案的 confidence 足够高:
     n ≥ 5 且 σ² < 0.05
     → "很确定这个是最好的"

  ④ 用户说"够了"或"直接做":
     → 立即停止迭代，进入仲裁引擎

时间预算:
  每轮迭代约 10~30 秒（取决于节点深度）
  简单任务总时间 < 2 分钟
  复杂任务总时间 < 5 分钟
```

## 树搜索 vs 旧版对比

```
旧版:
  方案A → Step1→Step2→Step3 → 一次性 V=0.88
  方案B → Step1→Step2→Step3 → 一次性 V=0.81
  → 1 轮，1 次评分，没有回头

新版:
  第1轮: 选择方案A→Step1-成功, 模拟V=0.85, 反传更新
  第2轮: 选择方案D(未探索), 模拟V=0.40, 反传更新
  第3轮: 选择方案B→Step1-成功, 模拟V=0.82, 反传更新
  第4轮: 选择方案A→Step1-失败, 模拟V=0.30, 反传更新 ← 探索了失败路径
  第5轮: 选择方案B→Step1-成功, 再次验证, 模拟V=0.80, 反传更新
  ...
  第8轮: 方案A V 趋于 0.84 不再变化 → 收敛 → 停止

区别:
  ✅ 多轮迭代，价值估计越来越准（旧版只评一次）
  ✅ UCB 驱动的探索-利用平衡（旧版只按顺序推）
  ✅ 探索了失败路径和备用方案（旧版只看成功路径）
  ✅ 反向传播让每个节点的价值都反映全局信息（旧版没有）
  ✅ 收敛判定防止过度推演（旧版永远推完所有方案）
