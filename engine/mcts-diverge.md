# 第1步: 发散引擎 — 八面镜 × 八路侦查 × 八面视角

> **一句话**: 以八面镜为统一框架，先识别领域、构造8个审视维度，再逐面侦查补全知识，最后每面生成1个方案视角。
> 八面镜覆盖万物——任何领域、任何任务，8个决策面不遗漏。

---

## 发散引擎全流程

```
LLM 理解用户需求 → 以八面镜为基准审视（代码仅给 hint，LLM 做最终判断）
    ↓ 每面映射为具体维度 + 自评 + 识别盲区
八路侦查（每面驱动1条侦查路径，盲区优先）
    ↓ 学习深度门禁
八面视角（每面生成1个方案视角）
    ↓ 8个跨视角方案
粗筛（仅方案 > 8时触发）
    ↓ 保留 4~8 个跨视角方案
输出给推演引擎
```

## 1.1 八面镜 — 抽象的决策审视框架

> 八面镜是**抽象类**——定义了任何决策都应该审视的8个面。
> 它不指定每个面"具体叫什么名字"，只定义"这个面问什么"。
> 
> 领域模板是**实现类**——八面镜在某领域的具体化。
> 比如面1"力量之源"在软件工程领域具体化为"技术栈"，在音乐领域具体化为"风格流派"。
> 
> LLM 应该**永远以八面镜的8个问题为基准**，领域模板只是帮助快速上手的参考，
> 不是必须遵守的规则。用户需求不同，同一个领域也可以有不同的具体化方式。

```
═══════════════════════════════════════════════════════════════
         八面决策框架 / Eight-Facet Decision Mirror
         
         抽象层（永远不变）              实现层（随需求变化）
         Abstract (invariant)           Concrete (context-dependent)
═══════════════════════════════════════════════════════════════

面1: 力量之源 / Source of Force
  抽象问: "这件事的驱动力从哪来？"
  实现为: [由用户需求决定——可能是技术栈、团队能力、个人经验、创作风格…]

面2: 根基承载 / Foundation & Capacity  
  抽象问: "什么是做这件事的底盘？"
  实现为: [由用户需求决定——可能是预算、架构、基础设施、知识储备…]

面3: 变动突破 / Change & Disruption
  抽象问: "哪里可能发生意想不到的变化？"
  实现为: [由用户需求决定——可能是需求变更、市场变化、突发风险、趋势转折…]

面4: 渗透传播 / Penetration & Diffusion
  抽象问: "怎么让效果真正渗透进去、扩散开来？"
  实现为: [由用户需求决定——可能是用户体验、传播渠道、教学方法、影响力…]

面5: 风险深渊 / Risk & Abyss
  抽象问: "最深的坑在哪？最坏会怎样？怎么绕过去？"
  实现为: [由用户需求决定——可能是安全漏洞、合规风险、创作瓶颈、认知陷阱…]

面6: 显眼依附 / Visible & Dependent  
  抽象问: "最引人注目的表面是什么？它底下依附于什么？"
  实现为: [由用户需求决定——可能是核心功能、亮点设计、关键证据、主推产品…]

面7: 边界止步 / Boundary & Limit
  抽象问: "有什么绝对不能碰的线？应该在哪里停下？"
  实现为: [由用户需求决定——可能是法律法规、道德底线、资源上限、身体极限…]

面8: 汇聚共赢 / Convergence & Mutual Benefit
  抽象问: "各方利益怎么平衡？有没有所有人都能接受的方案？"
  实现为: [由用户需求决定——可能是利益相关方、用户与团队、创作者与听众…]
═══════════════════════════════════════════════════════════════
```

### 使用方式

```
LLM 要做的事（按顺序）:

  ① 理解用户需求——他想做什么？在什么场景下？有什么约束？
  
  ② 以八面镜的8个抽象问题为基准，逐一审视这个需求
     不是"套模板"，而是"回答问题"
     
  ③ 对每一面，根据用户需求的具体内容，确定这一面的"实现"——
     这一面在这个需求里具体对应什么维度？叫什么名字？
     
  ④ 自评(0-10) + 证据 + 盲区识别
     盲区分类: `python scripts/mcts_compute.py classify-blindspot --score <0-10>`

领域模板只是参考，不是规则:
  - 如果用户需求恰好匹配某个预设领域 → 模板可以帮助快速上手
  - 如果用户需求跨领域/不在预设中 → 忽略模板，直接跟着八面镜走
  - 即使匹配了模板，也可以根据用户具体需求调整维度名称
  - 8个面全部审视，不跳过任何一个
```

## 1.2 八路侦查（领域自适应）

> 八面镜的每一面驱动一条侦查路径。不是写死的"项目代码/技术文档"，
> 而是每面问自己"这个面需要查什么？"——侦查路径随领域自动变化。

### 侦查路径生成规则

```
对于八面镜的每一面，LLM 确定该面的侦查路径:

  面1 力量之源 → 侦查: "谁在做这件事？他们的能力和经验？"
  面2 根基承载 → 侦查: "有什么可用资源？有什么硬性约束？"
  面3 变动突破 → 侦查: "环境在怎么变？有什么新趋势或意外因素？"
  面4 渗透传播 → 侦查: "信息/影响怎么传播？受众怎么接收？"
  面5 风险深渊 → 侦查: "类似的失败案例？最大的坑在哪？"
  面6 显眼依附 → 侦查: "最引人注目的部分是什么？它依赖于什么？"
  面7 边界止步 → 侦查: "有什么硬性规定/法律/道德底线？"
  面8 汇聚共赢 → 侦查: "各方利益诉求是什么？有没有共赢的先例？"

  每条路径的具体执行方式取决于领域:
    软件工程: 面1→查团队能力/技术栈, 面2→查项目约束/基础设施, ...
    音乐创作: 面1→查创作者风格/能力, 面2→查预算/设备/时间, ...
    医疗诊断: 面1→查医生专长/设备, 面2→查病史/检查条件, ...
```

### 学习深度门禁

```
八面审视和八路侦查完成后，进入方案生成前，必须通过学习深度门禁:

  `python scripts/mcts_compute.py check-learning-depth
     --domain <domain> --complexity <simple|medium|complex>
     --scores '{"面1":8,"面2":6,...}' --coverage '{"dimensions_covered":6,...}'`

  门禁检查:
    ① 8面是否都有评分？（不能有面被跳过）
    ② 盲区面是否已标注并制定了补资料计划？
    ③ 侦查路径是否覆盖了所有盲区面？
    ④ 关键信息是否有交叉验证？（至少2个来源）

  ✅ 通过 → 进入八面视角方案生成
  ⛔ 不通过 → 必须回到侦查阶段补全缺口，通过后才能继续
```

### 八路侦查输出格式

```
────────────────────────────
 【八路侦查报告】 任务: [xxx]

 面1 [维度名]: [侦查发现 + 来源 + 结论]
 面2 [维度名]: [侦查发现 + 来源 + 结论]
 ...
 面8 [维度名]: [侦查发现 + 来源 + 结论]

 交叉验证: (面X+面Y → 一致/矛盾 → 推荐)
 ────────────────────────────
```

## 1.3 八面视角（方案生成）

> 八面镜的每一面生成1个方案视角。不是独立的10种视角，而是八面的具体化。
> 每面问自己"从这个角度看，最好的方案是什么？"

### 视角生成规则

```
对于八面镜的每一面，生成1个方案视角:

  面1 力量之源 → "从主导力量的角度，最优方案是什么？"
  面2 根基承载 → "从资源约束的角度，最可行的方案是什么？"
  面3 变动突破 → "从应对变化的角度，最灵活的方案是什么？"
  面4 渗透传播 → "从传播影响的角度，最有效的方案是什么？"
  面5 风险深渊 → "从规避风险的角度，最安全的方案是什么？"
  面6 显眼依附 → "从核心亮点的角度，最吸引人的方案是什么？"
  面7 边界止步 → "从不可逾越边界的角度，最合规的方案是什么？"
  面8 汇聚共赢 → "从利益平衡的角度，最能让各方满意的方案是什么？"

  共8个方案视角，每个视角生成1个方案。
  如果某个面的方案与另一面高度重叠 → 合并，但保留两个面的标注。
```

### 方案之间必须有实质性差异

```
8个方案来自8个不同的审视面，天然有本质差异:

  ✅ 面1方案(力量主导) vs 面5方案(风险规避) — 完全不同视角
  ✅ 面2方案(资源约束) vs 面8方案(利益平衡) — 完全不同视角

  如果两个面的方案趋同 → 说明这两个面在该任务中高度相关
  → 合并方案但保留两个面的标注
  → 不凑数，有几个真正不同的方案就几个
```

### 每个方案的描述格式

```
  方案名称: [简短标识]
  八面对应: [面X: 维度名]
  一句话描述: [核心思路]
  来源依据: 八路侦查结论 + 知识图谱引用
  约束检查: 硬约束/软约束
  预期复杂度: [小/中/大]
```

## 1.4 粗筛（仅当方案数 > 8 时触发）

粗筛评分: `python scripts/mcts_compute.py rough-filter --solutions '<JSON>' --max-keep 5`
score = 可行性×0.5 + 成本效益×0.3 + (1-风险)×0.2, 保留top-3~5
