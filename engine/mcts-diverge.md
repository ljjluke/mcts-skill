# 第1步: 发散引擎 — 八面镜 × 八路侦查 × 八面视角

> **一句话**: 以八面镜为统一框架，先识别领域、构造8个审视维度，再逐面侦查补全知识，最后每面生成1个方案视角。
> 八面镜覆盖万物——任何领域、任何任务，8个决策面不遗漏。

---

## 发散引擎全流程

```
领域识别 → 构造8个审视维度（八面镜）
    ↓ 识别盲区维度
八路侦查（每面驱动1条侦查路径）
    ↓ 盲区驱动路径选择
八面视角（每面生成1个方案视角）
    ↓ 8个跨视角方案
粗筛（仅方案 > 8时触发）
    ↓ 保留 4~8 个跨视角方案
输出给推演引擎
```

## 1.1 八面镜 — 统一的决策审视框架

> 万物都可以从这8个决策面审视。任何领域、任何任务，这8个面都能覆盖。
> 这是整个发散引擎的顶层框架——维度构造、侦查路径、方案视角都由此派生。

```
═══════════════════════════════════════════════════════════════
         八面决策框架 / Eight-Facet Decision Mirror
         任何决策问题从这8个面照一遍，不遗漏
         Every decision, every domain — 8 facets, no blindspots
═══════════════════════════════════════════════════════════════

面1 (Facet 1): 力量之源 / Source of Force
  — 谁主导？有什么能力？理想目标是什么？
  — Who leads? What capabilities? What's the ideal outcome?
  Ask: "Where does the driving force come from?"

面2 (Facet 2): 根基承载 / Foundation & Capacity  
  — 需要什么资源？约束条件是什么？能承载吗？
  — What resources? What constraints? Can it bear the load?
  Ask: "What's the foundation this rests on?"

面3 (Facet 3): 变动突破 / Change & Disruption
  — 有什么变化因素？可能带来冲击还是机会？
  — What's changing? Threat or opportunity?
  Ask: "Where might the unexpected happen?"

面4 (Facet 4): 渗透传播 / Penetration & Diffusion
  — 信息/影响怎么流动？怎么让效果扩散？
  — How does influence spread? How to amplify impact?
  Ask: "How does this actually penetrate and reach people?"

面5 (Facet 5): 风险深渊 / Risk & Abyss
  — 最大的风险在哪？最坏情况？怎么避险？
  — What's the deepest pit? Worst case? How to avoid it?
  Ask: "Where's the deepest hole and how do we step around it?"

面6 (Facet 6): 显眼依附 / Visible & Dependent  
  — 最引人注意的是什么？背后依附什么？
  — What's most visible? What does it depend on underneath?
  Ask: "What's the shiny surface and what's holding it up?"

面7 (Facet 7): 边界止步 / Boundary & Limit
  — 有什么不可逾越的？应该在哪里停下来？
  — What cannot be crossed? Where must we stop?
  Ask: "Where's the line that absolutely cannot be crossed?"

面8 (Facet 8): 汇聚共赢 / Convergence & Mutual Benefit
  — 各方利益怎么平衡？有没有皆大欢喜的？
  — How to balance all interests? Is there a win-win?
  Ask: "Who benefits? Who loses? Can everyone win?"
═══════════════════════════════════════════════════════════════
```

### 八面镜的使用方式

```
八面镜不是"选6面用"，是8面全用。每一面都是一个独立的审视角度。

LLM 要做的事:
  ① 识别任务领域（代码辅助: identify-domain）
  ② 如果匹配到预设模板 → 直接加载该领域的8维映射
  ③ 如果没有预设模板 → 以八面镜为指导，主动构造8个具体维度
     构造规则: 每一面映射到该任务的具体维度，命名(2~4字)、描述、3个检测问题
  ④ 对每个维度自评(0-10) + 证据 + 盲区识别
  ⑤ 盲区分类: `python scripts/mcts_compute.py classify-blindspot --score <0-10>`
     完全空白(0-3)→补资料后才能生成 | 部分空白(4-6)→可生成但标注"待验证" | 已覆盖(7-10)→直接生成
```

### 领域识别

```
Step 0: 识别领域
  `python scripts/mcts_compute.py identify-domain --message "<用户消息>" --task "<任务描述>"`
  → mode="template": 有预设模板，直接加载
  → mode="dynamic": 无预设模板，LLM以八面镜为指导主动构造8维
```

## 1.2 八路侦查（领域自适应）

> 八面镜的每一面驱动一条侦查路径。不是写死的"项目代码/技术文档"，
> 而是每面问自己"这个面需要查什么？"——侦查路径随领域自动变化。

### 侦查路径生成规则

```
对于八面镜的每一面，LLM 确定该面的侦查路径:

  面1 力量之源 → 侦查: "谁在做这件事？他们的能力和经验？"
  面2 根基承载 → 侦查: "有什么可用资源？有什么硬性约束？"
  面3 变动突破 → 侦查: "环境在怎么变？有什么新趋势或意外因素？"
  面4 渗透传播 → 侦查: "信息/影响怎么传播？受众怎么接收？"
  面5 风险深渊 → 侦查: "类似的失败案例？最大的坑在哪？"
  面6 显眼依附 → 侦查: "最引人注目的部分是什么？它依赖于什么？"
  面7 边界止步 → 侦查: "有什么硬性规定/法律/道德底线？"
  面8 汇聚共赢 → 侦查: "各方利益诉求是什么？有没有共赢的先例？"

  每条路径的具体执行方式取决于领域:
    软件工程: 面1→查团队能力/技术栈, 面2→查项目约束/基础设施, ...
    音乐创作: 面1→查创作者风格/能力, 面2→查预算/设备/时间, ...
    医疗诊断: 面1→查医生专长/设备, 面2→查病史/检查条件, ...
```

### 学习深度门禁

```
八面审视和八路侦查完成后，进入方案生成前，必须通过学习深度门禁:

  `python scripts/mcts_compute.py check-learning-depth
     --domain <domain> --complexity <simple|medium|complex>
     --scores '{"面1":8,"面2":6,...}' --coverage '{"dimensions_covered":6,...}'`

  门禁检查:
    ① 8面是否都有评分？（不能有面被跳过）
    ② 盲区面是否已标注并制定了补资料计划？
    ③ 侦查路径是否覆盖了所有盲区面？
    ④ 关键信息是否有交叉验证？（至少2个来源）

  ✅ 通过 → 进入八面视角方案生成
  ⛔ 不通过 → 必须回到侦查阶段补全缺口，通过后才能继续
```

### 八路侦查输出格式

```
────────────────────────────
 【八路侦查报告】 任务: [xxx]

 面1 [维度名]: [侦查发现 + 来源 + 结论]
 面2 [维度名]: [侦查发现 + 来源 + 结论]
 ...
 面8 [维度名]: [侦查发现 + 来源 + 结论]

 交叉验证: (面X+面Y → 一致/矛盾 → 推荐)
 ────────────────────────────
```

## 1.3 八面视角（方案生成）

> 八面镜的每一面生成1个方案视角。不是独立的10种视角，而是八面的具体化。
> 每面问自己"从这个角度看，最好的方案是什么？"

### 视角生成规则

```
对于八面镜的每一面，生成1个方案视角:

  面1 力量之源 → "从主导力量的角度，最优方案是什么？"
  面2 根基承载 → "从资源约束的角度，最可行的方案是什么？"
  面3 变动突破 → "从应对变化的角度，最灵活的方案是什么？"
  面4 渗透传播 → "从传播影响的角度，最有效的方案是什么？"
  面5 风险深渊 → "从规避风险的角度，最安全的方案是什么？"
  面6 显眼依附 → "从核心亮点的角度，最吸引人的方案是什么？"
  面7 边界止步 → "从不可逾越边界的角度，最合规的方案是什么？"
  面8 汇聚共赢 → "从利益平衡的角度，最能让各方满意的方案是什么？"

  共8个方案视角，每个视角生成1个方案。
  如果某个面的方案与另一面高度重叠 → 合并，但保留两个面的标注。
```

### 方案之间必须有实质性差异

```
8个方案来自8个不同的审视面，天然有本质差异:

  ✅ 面1方案(力量主导) vs 面5方案(风险规避) — 完全不同视角
  ✅ 面2方案(资源约束) vs 面8方案(利益平衡) — 完全不同视角

  如果两个面的方案趋同 → 说明这两个面在该任务中高度相关
  → 合并方案但保留两个面的标注
  → 不凑数，有几个真正不同的方案就几个
```

### 每个方案的描述格式

```
  方案名称: [简短标识]
  八面对应: [面X: 维度名]
  一句话描述: [核心思路]
  来源依据: 八路侦查结论 + 知识图谱引用
  约束检查: 硬约束/软约束
  预期复杂度: [小/中/大]
```

## 1.4 粗筛（仅当方案数 > 8 时触发）

粗筛评分: `python scripts/mcts_compute.py rough-filter --solutions '<JSON>' --max-keep 5`
score = 可行性×0.5 + 成本效益×0.3 + (1-风险)×0.2, 保留top-3~5
