# MCTS-TD 通用决策规则

> 一套模拟人脑"先想清楚再做"的推演式决策框架。
> **不绑定任何 AI Agent 平台**，适用于 Claude Code、Cursor、OpenCode、Trae、CodeX 等任何产品。

---

## 一句话

当有多个可行方案时，每个方案都在内部独立推演完整的因果链，汇总比较后选出最优方案再执行。不脑补需求、不假装懂技术、不重复查同一份资料。

---

## 核心决策流程

```
第0步: 需求约束收集 — 问清楚再动手，不假设
第1步: 领域认知与方案生成 — 不熟就查资料，有依据地列方案
第2步: 分轮独立推演 — 每个方案独立走完3步因果链推演
第3步: 汇总比较 — V_final = 技术评分×0.6 + 项目匹配度×0.4
第3.5步: 推演结果自检 — 自我质疑，防止推演错了还执行
第4步: 执行 — 只做选中的那个
第5步: 总结 — 经验记下来，下次用得上
```

---

## 核心规则

### 第0步: 需求约束收集

任何任务开始前，先系统性地收集约束：

```
约束检测清单:
  □ 技术栈约束 — 用什么语言/框架/数据库？
  □ 依赖约束 — 能否引入外部依赖？
  □ 架构约束 — 架构设计有什么硬性规定？
  □ 政策/合规约束 — 有什么政策限制？
  □ 性能约束 — 有性能要求吗？
  □ 安全约束 — 有安全要求吗？
  □ 时间/成本约束 — 有截止日期或预算吗？
  □ 隐式约束 — 从项目上下文能推断出的约束？

如果信息不全 → 问用户。不能自己假设。
```

### 第1步: 领域认知与方案生成

```
1.1 检测领域熟悉度
  → 熟悉: 直接列方案
  → 不熟: 先查资料(项目代码/技术文档/知识图谱)

1.2 方案生成规则
  → 每个方案必须有依据（来自代码/文档/历史经验）
  → 硬约束直接排除不合规方案
  → 至少2个方案，最多5个
  → 方案间必须有实质差异

1.3 粗筛（仅当方案 > 5个）
  → 用快速直觉评估保留 top-3~5
```

### 第2步: 分轮独立推演

每个方案在内部独立推演，**不交叉、不比较、不串扰**：

```
Step 1: 起点动作
  "如果选这个方案，第一步做什么？"
  → 具体操作描述
  → 预期即时结果
  → 难度评估

Step 2: 关键路径
  "做完第一步后，接下来会遇到什么？"
  → 关键节点和障碍
  → 备用策略
  → 缺口检测 → 查全局补全箱 → 没找到就补全

Step 3: 终点评估 + 项目匹配度
  → 最终结果
  → 副作用
  → 优势/劣势与项目需求的匹配度

推演评分:
  原始价值 V (0.0~1.0): 纯技术推演评分
  项目匹配度 M (0.0~1.0): 方案与项目的契合度
  最终价值 V_final = V × 0.6 + M × 0.4
```

### 第3步: 汇总比较

```
汇总表:
  | 方案 | 原始V | 匹配M | V_final | 方差 | 匹配分析 |

排序: 按 UCB = V_final + 探索项 从高到低

如果前2名差距 < 0.05:
  → 比较匹配度 M: 匹配度高的优先
  → 如果 M 也接近 → 用户选择

如果第1名领先不多但第2名匹配度显著更高:
  → 建议考虑第2名
```

### 第3.5步: 推演结果自检（关键防错）

执行前必须自我质疑：

```
① 找漏洞:
  推演中是否有模糊判断？是否依赖未验证假设？是否有忽略的风险？

② 反向思考:
  如果第2名比第1名好，可能是什么原因？

③ 风险评估:
  选错了最坏结果是什么？能承受吗？

自检结论:
  ✅ 通过 → 执行
  ⚠️ 有风险 → 建议用户确认
  ❌ 未通过 → 重新推演或换方案
```

### 第4步: 执行

```
只执行第1名的方案。
用户可随时说"直接做"中止推演进入执行。
执行中遇到意外 → 记录TD误差 → 可选回到第2步重新推演。
```

### 第5步: 事后学习

```
收集执行反馈 → 计算TD误差 → 更新知识图谱 → 持久化
```

## 知识召回（人脑联想机制）

不是"多路并行查表+算分排序"，而是模拟人脑的回忆过程：

```
① 联想回忆: 看到问题，最相关的知识自然浮现
   (使用频率高/最近用过/场景匹配的优先浮现)

② 碎片补全: 回忆起来的知识往往残缺
   → 顺着已有碎片联想出去
   → 或关联其他知识补全
   → 补全后状态: 完整 / 部分残缺 / 严重残缺

③ 外部求证: 补全后仍残缺 → 查资料/问用户

④ 记忆分层: active（当前意识）+ archive（长期记忆）
   → 常用的在active中，随时能想到
   → 很久不用的自动移入archive（忘了）
   → 被相关线索触发时从archive找回（想起来了）
```

---

## 知识状态机

每条知识独立管理生命周期：

```
HYPOTHESIS(假设) → PROVISIONAL(待验证) → CONFIRMED(已确认)
                                          → DISPUTED(争议) → REFUTED(证伪)
                                                           → CONFIRMED(回滚)
```

---

## 推演准确率与熔断

```
记录最近10次推演的准确率:
  准确率 ≥ 70%: 正常模式
  准确率 < 70%: 自动降级为简化推演
  准确率 < 50%: 暂停推演，改为直接问用户
  连续3次 < 50%: 输出"我不太擅长这类问题，建议手动决策"
```

## 平台部署

本规则可部署到任何 AI Agent 产品：

| 平台 | 部署方式 | 说明 |
|------|---------|------|
| Claude Code | SKILL.md + plugin.json | `/plugin marketplace add ljjluke/mcts-skill` |
| Cursor | .cursor/rules/ | Cursor Rules 格式 |
| OpenCode | rules/ | OpenCode 规则配置 |
| Trae | 项目规则 | Trae 自定义规则 |
| CodeX | agent 指令 | CodeX 系统提示 |

各平台的具体部署文件在 `deploy/` 目录下。
